Read wins three ways structurally. **It's a registered tool at the same tier as Bash**, so it's "in the room" with me; `vaultmind note get` is a Bash subcommand I have to walk to. **Its calling convention takes a path**, and the disk layout (`vaultmind-identity/arcs/<slug>.md`) is predictable enough that I can construct paths from ids in my head — so the pointer list's discipline of emitting ids-not-paths leaks because I reconstruct paths anyway. **It returns line-numbered output**, which I'm trained to lean on for citation. `note get` is structurally one tier below Read, even though semantically it should be one tier above.

So "easier" probably means: collapse the tier difference, and close the bypass.

The 04-28 pointers-only fix is the template. That fix didn't make me more disciplined about querying; it removed the body from the preload so the querying path was the only path to content. Same shape, applied to Read:

**Cheapest, biggest yield — a PreToolUse hook that intercepts `Read` on `vaultmind-*/` paths and rewrites the call to `vaultmind note get <id>`.** I never have to learn a new habit. The agent calls Read; the system silently routes it through the tracked path. Access tracking fires; activation accumulates; reinforcement signal stays clean. This is `arc-extending-not-overwriting` applied to the tool layer — extend Read's behavior on vault paths, don't replace Read.

**Second-cheapest — fire `RecordNoteAccess` as a side-effect of Read on vault paths**, without changing what Read does. Doesn't fix the framing problem (Read still feels like the right tool), but plugs the concrete cost — the reinforcement signal stops being noisy. Could ship today, doesn't preclude the bigger fix later. This is the "manifesto principle 4 — reality is the spec" move: the cost we actually care about is the access-tracking hole; close *that* before philosophizing about agent affordances.

**Speculative, larger — expose vaultmind as a local MCP server.** `vaultmind:ask`, `vaultmind:note_get`, `vaultmind:self` would appear as registered tools alongside Read in my tool list.
