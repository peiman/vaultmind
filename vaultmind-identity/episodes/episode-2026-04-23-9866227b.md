## Episodic Layer v0 — plan

**Claim**: the smallest version that produces real data. No distillation yet. Capture only. Run it for ~2 weeks and let the substrate teach us what distillation should look like (manifesto 4).

**What is an episode**: one markdown file per Claude Code session. Durable, grep-able, plain text. Lives at `vaultmind-identity/episodes/YYYY-MM-DD-<sessionId>.md`.

**Fields** (derived mechanically from the session's JSONL transcript at `~/.claude/projects/<repo>/<sessionId>.jsonl`):
- `session_id`, `started_at`, `ended_at`, `cwd`, `git_branch`
- `commits_made`: hash + subject for commits created in-session
- `files_touched`: paths edited or written, with one-line purpose from the edit context
- `queries_run`: every `vaultmind ask` / `search` and the top hit
- `claims`: high-confidence assertions I made ("tests pass", "this is the root cause")
- `corrections`: Peiman's exact words when he corrected me, plus what I changed
- `surprises`: moments I explicitly said "I didn't expect X" or similar

**How**: a new `vaultmind episode` subcommand that parses a JSONL transcript and emits the markdown. Wired to Claude Code's `SessionEnd` hook via `.claude/hooks.json` so it runs automatically.

**What this is NOT** (scope discipline):
- Not embedded. Not indexed by vaultmind. Not surfaced by `ask`.
- Not distilled into arcs.
- Not activation-triggered. 
- Not a replacement for arcs — the raw substrate that arcs will be distilled *from* in the next step.

**Build order**:
1. Parse a completed JSONL transcript offline (dogfood today's session) — CLI subcommand, TDD. `vaultmind episode from-transcript <path>` → writes one `.md`.
2. Wire the SessionEnd hook once the parser is solid. 
3. Run for 2 weeks. Look at the corpus. Then plan distillation.

**Why this order**: building the parser as a standalone CLI means I can run it on any past transcript (there are dozens of JSONL files already) to backfill — I'll have a corpus to look at immediately, not just future sessions. That's faster feedback than waiting two weeks of new sessions.

**TDD first steps**: write a test that feeds a small fixture JSONL and asserts the output markdown has the expected fields populated. Then write the parser. Then wire the hook.
