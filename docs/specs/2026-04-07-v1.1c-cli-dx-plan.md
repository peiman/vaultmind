### Task 1: Document wikilink convention

**Files:**
- Create: `docs/wikilink-convention.md`

This is docs-only — do it first since it's referenced by other tasks.

- [ ] **Step 1: Create the document**

Create `docs/wikilink-convention.md`:

```markdown
# Wikilink Convention

VaultMind vault notes must use Obsidian-compatible wikilink format.

## The Rule

Always use `[[filename|Display Text]]`, never `[[Title Case]]`.

**Correct:**
- `[[context-pack|Context Pack]]` — links to `context-pack.md`, displays "Context Pack"
- `[[rag|RAG]]` — links to `rag.md`, displays "RAG"
- `[[source-tulving-1972|Tulving 1972]]` — links to source note

**Wrong:**
- `[[Context Pack]]` — Obsidian can't find `context-pack.md` by title
- `[[RAG]]` — creates a new empty "RAG.md" instead of linking to `rag.md`

## Why

Obsidian resolves `[[target]]` by matching `target` against filenames (without `.md`).
VaultMind resolves by ID, title, alias, AND filename stem.
Using the filename format works in both systems.

## CLI Command References

CLI commands are NOT vault notes. Use backtick code format, not wikilinks:

**Correct:** The `memory recall` command traverses the graph.
**Wrong:** The [[memory-recall|memory recall]] command traverses the graph.

## Detection

Run `vaultmind doctor` to detect incompatible links:

```bash
vaultmind doctor --vault your-vault
```

Doctor reports `Obsidian-incompatible links` with suggested fixes showing the correct `[[filename|Display]]` format.

## Enforcement

Use `vaultmind lint --fix-links` (v1.1+) to bulk-fix all incompatible wikilinks across the vault.
```

- [ ] **Step 2: Commit**

```bash

git commit -m "docs: add wikilink convention guide for vault authors"
```

### Task 3: Add `vaultmind lint --fix-links`

**Files:**
- Create: `cmd/lint.go`
- Create: `internal/mutation/lint.go`
- Create: `internal/mutation/lint_test.go`
- Modify: `internal/config/commands/` (new lint config)

This is the largest task. The lint command:
1. Reads all .md files in the vault
2. For each `[[Target]]` wikilink in the body (not frontmatter), checks if Target matches a filename stem
3. If not, looks up the note by title/alias via the index to find the filename
4. With `--dry-run` (default): shows what would change
5. With `--fix`: applies changes

- [ ] **Step 1: Read the existing doctor incompatible-link logic**

The doctor already detects these links. Lint reuses the detection but adds file rewriting.

- [ ] **Step 1: Check existing hook setup**

Read `.lefthook.yml` or `.claude/hooks.json` or whatever hook system the project uses.

- [ ] **Step 2: Add doctor check**

Add a pre-commit hook that runs:
```bash
vaultmind doctor --vault vaultmind-vault --json | python3 -c "
import sys,json
d=json.load(sys.stdin)
issues = d['result']['issues']
if issues['obsidian_incompatible_links'] > 0:
    print(f'ERROR: {issues[\"obsidian_incompatible_links\"]} Obsidian-incompatible links found')
    sys.exit(1)
"
```

This is optional/best-effort — if vaultmind binary isn't built, the hook should skip gracefully.
