## Overview

## P1.1a — Bug Fixes & Correctness

5 items. No dependencies. Can be implemented immediately.

### A1. Tilde-fence stripping in alias mention scanner

**Problem:** `StripForAliasMatch` in `inferred.go:11` uses `codeFenceRe` that only matches triple-backtick fences, not `~~~` tilde fences. The parser handles both. Aliases inside tilde-fenced code blocks produce false `alias_mention` edges.

**Fix:** Extend `codeFenceRe` to match both:
```go
codeFenceRe = regexp.MustCompile("(?s)(```[^`]*```|~~~[^~]*~~~)")
```

**Test:** Add a note with an alias inside a tilde fence. Verify no `alias_mention` edge is created.

**Files:** `internal/index/inferred.go`, `internal/index/inferred_test.go`

---

### A2. Tag overlap edges bidirectional

**Problem:** `ComputeTagOverlap` inserts edges with `p.a < p.b` only. `links out A` shows B but `links out B` doesn't show A. `memory related` and `context-pack` query both directions (unaffected), but `links out` is asymmetric.

**Fix:** Insert two rows per qualifying pair: `(p.a, p.b)` and `(p.b, p.a)`.

**Test:** Create two notes sharing rare tags. Verify `links out` from both directions returns the other.

**Files:** `internal/index/inferred.go`, `internal/index/inferred_test.go`

---

### A3. Alias map collision for normalized duplicates

**Problem:** In `ComputeAliasMentions`, `aliasToNoteID` uses `strings.ToLower(text)` as key. Two aliases normalizing to the same lowercase string (e.g., "API" from different notes) collide — last one wins non-deterministically.

**Fix:** Use `map[string][]string` and insert edges for all matching notes:
```go
aliasToNoteIDs := make(map[string][]string)
// ...
aliasToNoteIDs[strings.ToLower(e.text)] = append(aliasToNoteIDs[...], e.noteID)
```

**Test:** Create two notes with aliases "API" and "api". Verify both get `alias_mention` edges when a third note mentions "api".

**Files:** `internal/index/inferred.go`, `internal/index/inferred_test.go`

---

### A4. Context-pack caches QueryFullNote results

**Problem:** `contextpack.go` calls `QueryFullNote` during frontmatter packing (line 119) and again during body backfill (line 156) for the same note. At 50 items, that's 500 queries instead of 250.

**Fix:** Add a `map[string]*FullNote` cache in `ContextPack`:
```go
noteCache := make(map[string]*index.FullNote)
// helper:
loadNote := func(id string) (*index.FullNote, error) {
    if cached, ok := noteCache[id]; ok { return cached, nil }
    full, err := db.QueryFullNote(id)
    if err != nil { return nil, err }
    noteCache[id] = full
    return full, nil
}
```

**Test:** Verify context-pack produces identical output with cache (correctness check, not performance).

**Files:** `internal/memory/contextpack.go`, `internal/memory/contextpack_test.go`

---

### A5. Doctor flags `_path:` pseudo-ID resolutions

**Problem:** Links like `[[Memory Recall]]` resolve to `_path:Memory Recall.md` (a non-existent file) and are counted as "resolved." Doctor should flag these as warnings.

**Fix:** Add a diagnostic query:
```sql
SELECT src_note_id, dst_note_id, dst_raw FROM links
WHERE resolved = TRUE AND dst_note_id LIKE '_path:%'
```
Add to `DoctorIssues` as `PathPseudoIDLinks int` with details.

**Test:** Insert a link with `dst_note_id = '_path:Foo.md'`. Verify doctor flags it.

**Files:** `internal/query/doctor.go`, `internal/query/doctor_test.go`, `cmd/doctor.go`

## P1.1b — Error Handling

3 items. No dependencies.

### B1. `index` command vault errors through JSON envelope

**Problem:** `cmd/index.go` validates the vault path directly (line 29-31) and returns raw text even with `--json`. It doesn't use `OpenVaultDB` because it creates an indexer directly.

**Fix:** Check `--json` flag and use `cmdutil.WriteJSONError` for the vault path validation and config loading errors. Reuse the `ErrAlreadyWritten` sentinel pattern.

**Files:** `cmd/index.go`

---

### B2. Granular error codes

**Problem:** All vault-opening failures get `code: "vault_error"`. Agents can't distinguish path-not-found from database-locked from config-parse.

**Fix:** In `OpenVaultDB`, detect error type and set specific codes:
```go
if os.IsNotExist(err) → "vault_not_found"
if strings.Contains(err.Error(), "locked") → "database_locked"
if strings.Contains(err.Error(), "config") → "config_error"
default → "vault_error"
```

Pass the code through `OpenVaultDBOrWriteErr`.

**Test:** Test each error class returns the correct code.

**Files:** `internal/cmdutil/helpers.go`, `internal/cmdutil/helpers_test.go`

---

### B3. Duplicate ID behavior — refuse or document

**Problem:** Session 03 recommended "refuse the second insert." Current behavior: logs warning, increments counter, overwrites. The behavior is ambiguous — neither fully refusing nor explicitly documented as intentional.

**Fix:** Change `INSERT OR REPLACE` to `INSERT OR IGNORE` for the second file. Log warning with both file paths. Add the conflict to `IndexResult.Warnings` as a structured issue:
```go
type IndexWarning struct {
    Code    string `json:"code"`
    Message string `json:"message"`
    Paths   []string `json:"paths,omitempty"`
}
```

The first file wins. The second file is skipped with a warning.

**Test:** Index two notes with the same `id`. Verify only the first is stored and a warning is emitted.

**Files:** `internal/index/indexer.go`, `internal/index/indexer_test.go`

## P1.1c — CLI & Developer Experience

4 items. No dependencies.

### C1. `vaultmind lint --fix-links`

**Problem:** No bulk command to rewrite wikilinks from `[[Title]]` to `[[filename|Title]]`. During dogfooding, a custom Python script was needed to fix 138 links.

**Fix:** Add a `lint` command (or `doctor --fix`) that:
1. Reads all .md files in the vault
2. For each `[[Target]]` wikilink in the body, checks if Target matches a filename stem
3. If not, but it matches a note title, rewrites to `[[filename|Target]]`
4. With `--dry-run`, shows what would change without writing
5. With `--fix`, applies changes

**Files:** New `cmd/lint.go`, new `internal/mutation/lint.go`, new config registration.

---

### C2. `note create --stdin`

**Problem:** No way to pipe body content from an agent into a new note. `note create` works with templates but doesn't accept stdin for the body.

**Fix:** Add `--body-stdin` flag to `note create`. When set, read body content from stdin after template processing. The template provides frontmatter; stdin provides body:
```bash
echo "## Overview\n\nThis is the body." | vaultmind note create --type concept --title "New Concept" --body-stdin
```

**Files:** `cmd/note_create.go`, `internal/template/process.go`

---

### C3. Document wikilink convention

**Problem:** No documentation explains the `[[filename|Display Text]]` convention or when to use backtick-code for CLI commands.

**Fix:** Add a `docs/wikilink-convention.md` explaining:
- Always use `[[filename|Display Text]]` — Obsidian resolves by filename
- CLI commands use backtick-code (`` `memory recall` ``), not wikilinks
- Run `doctor` to catch incompatible links
- Use `lint --fix-links` to bulk-fix

**Files:** `docs/wikilink-convention.md`

---

### C4. Pre-commit hook: doctor check

**Problem:** Incompatible wikilinks can be committed without detection.

**Fix:** Add a pre-commit hook (via lefthook, already in the project) that runs:
```bash
vaultmind doctor --vault vaultmind-vault --json | jq '.result.issues.obsidian_incompatible_links'
```
Fail if the count is > 0.

**Files:** `.lefthook.yml` or equivalent hook config

## P1.1d — Performance

2 items. No dependencies.

### D1. `QueryFullNote` single JOIN query

**Problem:** `QueryFullNote` fires 5 sequential queries per note (frontmatter, body, tags, links, aliases). At bulk-read scale (memory recall with 50 nodes), this is 250 queries.

**Fix:** Replace with a single query that JOINs all tables, or use 2 queries: one for the note row + one aggregating related data via GROUP_CONCAT or JSON_GROUP_ARRAY (SQLite JSON1 extension).

**Test:** Benchmark before/after. Verify identical output.

**Files:** `internal/index/queries.go`

---

### D2. Path-based exclude matching

**Problem:** `.vaultmindignore` patterns match on filename only. Directory subtree exclusions (e.g., `archive/`) don't work.

**Fix:** Use `filepath.Match` or `doublestar` glob matching against the full relative path, not just the filename.

**Test:** Create a vault with `archive/note.md`. Add `archive/` to `.vaultmindignore`. Verify the note is excluded.

**Files:** `internal/vault/scanner.go`

---

## P1.1e — Retrieval Architecture

5 items. Dependencies: E1 before E2-E4. E5 depends on E2+E3.

### E1. Wire FTSRetriever into RunSearch

**Problem:** The `Retriever` interface and `FTSRetriever` exist but `RunSearch` still calls `SearchFTS` directly.

**Fix:** Change `RunSearch` to accept a `Retriever`:
```go
func RunSearch(retriever query.Retriever, cfg SearchConfig, w io.Writer) error {
    results, total, err := retriever.Search(ctx, cfg.Query, cfg.Limit, cfg.Offset, filters)
    // ...
}
```
`cmd/search.go` constructs `&query.FTSRetriever{DB: vdb.DB}` and passes it.

`SearchResult.Hits` changes type from `[]index.FTSResult` to `[]query.ScoredResult`.

**Files:** `internal/query/search.go`, `cmd/search.go`

---

### E2. Cross-query score normalization

**Problem:** Min-max normalization is per-query relative. Score 0.7 from two different queries means different things.

**Fix:** Add a sigmoid normalization option to `FTSRetriever`:
```go
normalized = 1 / (1 + math.Exp(-k * (score - median)))
```
With `k` tuned to the corpus's BM25 distribution. Requires a one-time calibration step during indexing that computes percentile boundaries.

Alternative: log-normalize raw BM25 scores before min-max, which dampens outliers and makes scores more stable across queries.

**Files:** `internal/index/fts.go` or `internal/query/fts_retriever.go`

---

### E3. Add EmbeddingRetriever

**Problem:** No semantic/embedding-based retrieval. FTS is keyword-only.

**Prerequisite:** Schema migration adding `embedding BLOB` column (P1.1f item F1).

**Fix:** Implement `EmbeddingRetriever` satisfying the `Retriever` interface:
```go
type EmbeddingRetriever struct {
    DB        *index.DB
    Embedder  Embedder // interface for embedding provider
}

type Embedder interface {
    Embed(ctx context.Context, text string) ([]float32, error)
}
```
Uses cosine similarity against stored embeddings. Returns normalized scores.

**Files:** `internal/query/embedding_retriever.go`, new `internal/query/embedder.go` interface

---

### E4. BM25 score normalization — query-independent

**Problem:** Min-max is relative to each result set. Need absolute scoring for blending.

**Fix:** During indexing, compute corpus-wide BM25 statistics (mean, stddev across a sample of common queries). Store in a metadata table. Use z-score normalization at query time:
```
normalized = sigmoid((raw_bm25 - mean) / stddev)
```

This produces query-independent scores in [0,1] that can be blended with embedding similarity.

**Files:** `internal/index/fts.go`, schema migration for stats table

---

### E5. Add HybridRetriever

**Problem:** No way to combine FTS and embedding scores.

**Prerequisite:** E2 (FTSRetriever wired in), E3 (EmbeddingRetriever).

**Fix:** Implement `HybridRetriever`:
```go
type HybridRetriever struct {
    FTS       *FTSRetriever
    Embedding *EmbeddingRetriever
    Alpha     float64 // weight: 0=pure FTS, 1=pure embedding
}
```
Runs both retrievers, merges results by weighted score combination, deduplicates by note ID.

**Files:** `internal/query/hybrid_retriever.go`

---

## P1.1f — Schema & Data

4 items. F1 before F2. F3 independent. F4 independent.

### F1. Schema migration 003: embedding column

**Fix:** Add goose migration:
```sql
-- +goose Up
ALTER TABLE notes ADD COLUMN embedding BLOB;
CREATE INDEX IF NOT EXISTS idx_notes_embedding ON notes(id) WHERE embedding IS NOT NULL;
```

**Files:** `internal/index/migrations/003_add_embedding_column.sql`

---

### F2. Embedding computation during indexing

**Problem:** No infrastructure to compute and store embeddings.

**Fix:** Add an optional `--embed` flag to `index` that calls an `Embedder` for each note's body text and stores the result. Default: skip embedding (backward compatible).

Config: `app.index.embed_model` (e.g., `"openai:text-embedding-3-small"` or `"local:all-MiniLM-L6-v2"`).

**Files:** `internal/index/indexer.go`, `cmd/index.go`, new `internal/index/embed.go`

---

### F3. Access frequency tracking

**Problem:** No `last_accessed_at` or `access_count` on notes. Prerequisite for ACT-R-style decay scoring in context-pack.

**Fix:** Schema migration:
```sql
-- +goose Up
ALTER TABLE notes ADD COLUMN last_accessed_at TEXT;
ALTER TABLE notes ADD COLUMN access_count INTEGER NOT NULL DEFAULT 0;
```

Update `QueryFullNote` and `note get` to increment `access_count` and set `last_accessed_at` on read.

**Files:** Migration file, `internal/index/queries.go`, `cmd/note_get.go`

---

### F4. `context-pack --depth N`

**Problem:** Context-pack only considers depth-1 edges (direct neighbors). An agent wanting "most important notes in a 3-hop neighborhood, packed into my budget" must combine `recall` + manual selection.

**Fix:** Add `--depth` flag (default 1). Use BFS traversal (reuse `graph.Traverse`) to collect candidates up to depth N, then sort by priority and pack into budget.

**Files:** `internal/memory/contextpack.go`, `cmd/memory_context_pack.go`, config registration

## P1.1g — Testing

1 item. No dependencies.

### G1. Migrate remaining tests to fixture vault

**Problem:** Multiple tests still reference `../../vaultmind-vault` and break when vault content changes.

**Fix:** Audit all test files for `testVaultPath = "../../vaultmind-vault"`. For each:
- If the test needs specific note content → create a minimal fixture in `test/fixtures/testvault/`
- If the test builds its own temp DB → no change needed (already isolated)
- If the test checks vault-wide statistics → make assertions relative (e.g., `> 0`) not absolute (e.g., `== 42`)

**Files:** All `*_test.go` files referencing `testVaultPath`, `test/fixtures/testvault/`

## P1.1h — Memory Features

1 item. No dependencies.

### H1. `memory summarize` and reflection note type

**Problem:** No mechanism for agents to create higher-order knowledge by reflecting on retrieved content. The SRS defined a `reflection` note type and `memory summarize` command but both were deferred.

**Fix:**
1. Add `reflection` to the type registry with fields: `source_ids` (notes that prompted the reflection), `trigger` (what question led to the reflection), `insight` (the synthesized knowledge).
2. Add `memory summarize` command that takes a set of note IDs, retrieves their content, and generates a summary note of type `reflection`.
3. During context-pack, give reflection notes elevated priority (between explicit_relation and explicit_link).

**Files:** Config registration for reflection type, `cmd/memory_summarize.go`, `internal/memory/summarize.go`, update `edgePriority` in `contextpack.go`
