# Park et al. — Generative Agents: Interactive Simulacra of Human Behavior (2023)

Park et al. (Stanford) introduce a multi-agent simulation environment (Smallville) in which 25 LLM-powered agents live in a virtual town, form relationships, make plans, and remember past events. The paper's memory architecture — the memory stream — is the most cited prior art for LLM agent memory systems.

Each stored memory receives a **retrieval score** computed as a weighted sum of three factors:

    score = alpha_recency * recency + alpha_importance * importance + alpha_relevance * relevance

- **Recency**: Exponential decay over wall-clock hours since the memory was last accessed. Agents forget quickly by default.
- **Importance**: A 1-10 score generated by asking the LLM "how important is this event?" at storage time.
- **Relevance**: Cosine similarity between the memory's embedding and the current query embedding.

All three components are normalized to [0,1] before weighting. The authors set all alpha values to 1.0 (equal weighting) but note these could be tuned.

VaultMind's three-factor activation model parallels this structure but replaces each component with a stronger theoretical foundation. Recency uses ACT-R power-law decay (empirically validated) instead of exponential decay (ad hoc). Importance is replaced by `access_count`-based storage strength (Bjork & Bjork, 1992) rather than LLM-generated scores (expensive and inconsistent). Relevance uses the same cosine similarity approach via BGE-M3 embeddings. The session-aware time compression in [[concept-temporal-activation-for-intermittent-systems|Temporal Activation]] further improves recency for intermittent CLI usage, which Park et al. do not address.

The Smallville simulation is also notable for demonstrating **reflection**: agents periodically generate higher-level insights by synthesizing patterns across recent memories. This is the generative agent analog to VaultMind's `vault_ask` — synthesis over retrieved context.

The paper's memory stream maps naturally onto Obsidian's append-only daily notes, reflection maps onto periodic summarization passes, and the three-factor retrieval scoring formula directly informs VaultMind's [[concept-base-level-activation|Base-Level Activation]] scoring approach.
