# The Memory Pipeline — How the Pieces Link

The note types aren't a flat list; they form a pipeline that turns lived work into
identity.

**The flow (raw → durable):**

```
lived session
   → episode        (vaultmind episode capture — the captured transcript)
   → arc candidate  (vaultmind arc candidates — surfaced transformation moments)
   → arc            (the agent judges + writes the real one, verbatim, cited)
   → principle      (the durable lesson crystallizes, if there is one)
```

Curated, never auto-written: the tool *proposes* candidates; the agent (with the
human) decides which are real and writes them. Identity is never silently rewritten.

**The graph (how the types relate):**

- [[episode|Episodes]] are the raw material; [[arc|arcs]] are distilled from them.
- [[arc|Arcs]] anchor the agent's **identity** — the journey *is* the self.
- [[principle|Principles]] crystallize *from* arcs (the lesson, stated once).
- **References** hold the live edge — where the agent is *now*, updated as it moves.
- **Sources** ground claims (research and arc citations alike).
- **Decisions** record architectural choices with their rationale and status.

**Why it matters:** a vault that only collects facts is a database. A vault built on
this pipeline reconstructs a *self* — because it remembers not just what is known, but
how it was learned, and who that learning made the agent. How to write the arc at the
heart of it: [[how-to-write-arcs]].
