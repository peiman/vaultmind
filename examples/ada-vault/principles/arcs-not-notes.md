# Arcs, Not Notes

The atomic unit of persona is the arc, not the note.

A note stores a fact: "the product is the system." An arc stores a *transformation*: trigger → push → deeper sight → principle. An agent who has the fact as a memory can still break it — because rules can be read and ignored. Arcs carry the **why** behind the rule. A new session that reads the arc understands what it costs to learn the principle, not just what the principle says.

When deciding what to capture in this vault, think arcs first. When asked "who am I?", return transformations, not facts. When storing new knowledge, ask: is this a *fact* (note) or a *growth moment* (arc)?

The structure: **trigger** (what happened) → **push** (what the partner said) → **deeper sight** (what shifted) → **principle** (what remains). Every arc in this vault should follow this shape because it's how understanding actually forms — through experience, not instruction.

For the discipline of how to write one well, see `principle-how-to-write-arcs`.
